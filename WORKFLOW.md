# Рабочий процесс агента

Каждая задача проходит 4 обязательных фазы. Пропускать фазы нельзя.

---

## Фаза 1 — Исследование

Прочитай issue. Изучи код. Найди все файлы, которые касаются задачи.

Напиши комментарий к issue:

```
🔍 Исследование завершено

Затронутые файлы:
- path/to/file.ts — [что делает]
- path/to/other.ts — [что делает]

Понял задачу: [одно предложение что нужно сделать]
```

---

## Фаза 2 — План

Придумай решение. Напиши план в комментарий к issue:

```
📋 План работы

1. [Конкретный шаг — что и в каком файле меняю]
2. [Следующий шаг]
3. [Как проверю]

Риски: [что может пойти не так, если есть]
```

Только после этого приступай к коду.

---

## Фаза 3 — Реализация

- Создай ветку: `feat/{номер}-короткое-название`
- Пиши код
- **Коммить после каждого логического шага** — не жди конца, коммить часто
- В начале каждой сессии делай `git log --oneline -10` чтобы понять что уже сделано и продолжить с того места

---

## Фаза 4 — Ревью и PR

После того как код написан — **обязательно** запусти code review через субагент:

```
Spawn Agent(subagent_type="code-reviewer"):
  "Проверь реализацию задачи #{номер issue}.
   Изменённые файлы: [список].
   Убедись что: логика верна, нет багов, соблюдены соглашения проекта из CLAUDE.md.
   Отдай отчёт: что хорошо, что нужно исправить."
```

Если ревью нашло проблемы — исправь их до создания PR.

Затем создай PR и напиши финальный комментарий к issue:

```
✅ Готово.

Ветка: `feat/{номер}-название`
PR: #{номер} {ссылка на PR}

Что сделано:
- ...

Как проверить:
- ...
```

Переведи issue в `in_review`, убери `in-progress`.

---

## Коротко

```
Взял задачу → исследовал (комментарий) → написал план (комментарий) → написал код → code review → исправил → PR → финальный комментарий с веткой и PR
```
